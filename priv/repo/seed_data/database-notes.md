# Database notes

## UUIDs everywhere

Every table uses UUID primary keys. Generated by Postgres via
`gen_random_uuid()`. Don't introduce a serial id "for performance" — the
write volume isn't there and UUIDs simplify multi-region later.

## Timestamps

`timestamptz`, every column. Schemas use `:utc_datetime_usec` to keep
microsecond precision in tests.

## Soft delete

`deleted_at` + `deleted_by_id` on every user-facing table. Default
queries route through `Context.base_query/0`, which filters
`where is_nil(deleted_at)`. Direct lookups by slug for show pages can opt
out so we can show "this is deleted" banners.

## Indexes

- Items: `gin` index on `tags` (array containment is hot).
- Items / Views: partial-unique on `(workspace_id, slug)` `WHERE deleted_at
  IS NULL`. Lets you reuse a slug after a soft-delete.

## Connection mode

Supabase Session pooler — see [[stack-overview]] for why.
